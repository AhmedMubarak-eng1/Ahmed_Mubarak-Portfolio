Dual-Tank Pump Control System – Allen-Bradley PLC & HMI

McMaster University – Lab Project 2024–2025 Allen-Bradley CompactLogix FactoryTalk View HMI

Overview

Designed and programmed a complete dual-tank pump control system using an Allen-Bradley CompactLogix PLC (Studio 5000) and FactoryTalk View HMI. The system manages water levels across two interconnected tanks by orchestrating four pumps and four solenoid valves — balancing flow, protecting equipment, and giving operators full visibility and control through a professional operator interface.

The architecture was built around Add-On Instructions (AOIs) — reusable, encapsulated logic blocks that standardize how every pump and valve is controlled, monitored, and protected. This mirrors the modular, scalable PLC programming approach used on real industrial projects.

Add-On Instruction (AOI) Architecture

Rather than writing individual rungs for each pump and valve, the program was structured using custom Add-On Instructions — a professional-grade programming technique that encapsulates logic into reusable, self-contained blocks with defined parameters, internal tags, and their own logic routines.

AOI_Pump
Controls each pump across Auto, Manual, and Maintenance modes. Handles operator start/stop, program-mode commands, feedback monitoring, fault detection, fault-ignore capability, and maintenance restart logic. Instantiated for P101, P102, P103, P104.
AOI_Valve
Manages each solenoid valve across Auto, Manual, and Maintenance modes. Tracks open/close feedback, issues close commands, monitors fault conditions, and supports maintenance start and fault-ignore. Instantiated for V100, V101, V102, V103.

Both AOIs use User-Defined Data Types (UDTs) for structured parameter passing and internal state management, keeping the MainRoutine clean and the logic easy to audit, modify, and scale.

Control Logic & PLC Program

  • Level-based pump control: Tank level setpoints (high/low) automatically trigger pump start/stop commands — Tank 1 feeds Tank 2 when levels demand, and Tank 2's transfer pump activates based on downstream level targets
  • Pump alternation logic: Pumps rotate lead/lag assignments to balance runtime hours across all four units, reducing wear and extending equipment life — a direct reflection of industry-standard pump sequencing practice
  • Maintenance mode: Any individual pump or valve can be taken offline for maintenance without stopping the rest of the system — the control logic automatically reassigns duties to available equipment
  • Fault handling & interlocks: Each AOI monitors its device for faults. Detected faults latch an alarm, surface to the HMI, and prevent unsafe operation. Operators can acknowledge and ignore non-critical faults via the Fault_Ignore parameter without clearing the program state
  • Auto/Manual/Maintenance mode selection: Every pump and valve supports three independent operating modes — switchable from the HMI per device, giving operators granular control without program modifications
  • Simulation inputs: A dedicated simulation mode allows level inputs and device feedback to be forced in software, enabling full system testing and HMI validation without physical hardware

HMI Features

  • Animated P&ID overview screen: Graphical representation of both tanks with live level readouts, animated pump and valve symbols that change color and state in real time (green = running/auto, grey = off, orange = maintenance)
  • Per-device mode control popups: Dedicated screens for pump and valve mode selection, allowing operators to switch any device between Auto, Manual, and Maintenance without navigating away from the overview
  • Full control panel screen: Individual Manual Start, Manual Stop, Maintenance Start, Pump Ready indicator, and Auto Stop buttons for each of the four pumps — and Manual Open, Manual Close, Valve Ready, Maintenance Start, and Auto Stop for each valve
  • Fault ignore buttons: P1–P4 and V0–V3 Ignore Fault buttons accessible directly on the overview screen, allowing operators to acknowledge non-critical device faults without clearing the underlying alarm
  • Live alarm banners: Active alarms (e.g. T2 LOW LEVEL) appear as color-coded banner alerts on the main screen, immediately drawing operator attention to abnormal conditions
  • Tank level display: Real-time numeric level readouts for Tank 1 and Tank 2 in the header bar, with alarm state highlighting when levels breach high or low thresholds
  • Simulation mode: Sim. Inputs and Simulation buttons on the main screen allow engineers to force inputs and simulate device states for testing without physical process changes

Technical Highlights

Platform

Allen-Bradley CompactLogix PLC programmed in Studio 5000 using Ladder Diagram (LD) and Structured Text (ST). Program organized into MainTask → MainProgram → MainRoutine with separate AOI Logic routines for each device type.

AOI Design

Custom AOI_Pump and AOI_Valve Add-On Instructions with UDT-based parameters, internal local tags, and self-contained logic routines — enabling clean, scalable, and auditable code that mirrors industrial standards.

HMI

FactoryTalk View ME with P&ID-style animated graphics, multi-screen navigation, per-device mode popups, live alarm banners, fault ignore controls, and a dedicated simulation interface for testing.

Alternation & Maintenance

Pump lead/lag alternation logic balances runtime across all four units. Any pump or valve can enter Maintenance mode independently while the control system reroutes duties to remaining available equipment.

Results & Impact

Delivered a fully functional, production-grade dual-tank pump control system demonstrating professional PLC architecture, reusable AOI design, and comprehensive HMI development. The use of Add-On Instructions produced modular, maintainable code that could scale to additional pumps or tanks with minimal rework. Pump alternation and maintenance mode logic operated correctly across all test scenarios, and the FactoryTalk View HMI gave operators clear, real-time control and alarm visibility. The project directly applied industrial-standard PLC programming practices — mirroring the control architecture used in water treatment, oil & gas, and process manufacturing facilities.
